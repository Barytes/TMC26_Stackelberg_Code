# STAGE1 Convergence Alignment Report (2026-03-09)

## 1) What was inconsistent

Observed issues from prior runs:
- **dist-to-SE was not consistently aligned** across scripts/variants:
  - Some trajectories used `SearchStep.dist_to_se` coming from in-algorithm proxy logic.
  - Some comparisons used separately computed final-point distance.
  - For `n_users > 16`, algorithm-side SE proxy could stay unavailable (`NaN`) while diagnostics still proceeded.
- **Figure/report linkage drift**:
  - Figure-style diagnostics and A/B text summaries were generated by different scripts with different metric plumbing.
  - This made it possible to compare values that were not guaranteed to share the same oracle/domain/instance.

## 2) What was fixed

### A. Strict distance metric alignment + hard failure checks

Updated `scripts/run_stage1_boundary_visualization.py`:
- Compute **single true-SE proxy** once per trial using `baseline_stage1_grid_search_oracle(...)` on the exact sampled users.
- Recompute trajectory `dist_to_se` from `(pE,pN)` to that same `true_se` for every step/final point.
- Add strict checks:
  - fail if `true_se` is non-finite
  - fail if any trajectory `dist_to_se` is NaN/Inf

Result: `algorithm5_diagnostics_trial0.txt` now has finite distance progression (no NaN).

### B. Stronger Stage-I search upgrade (beyond estimator tweak)

Updated `src/tmc26_exp/stackelberg.py`:
- Added `_perturb_restart_candidate(...)`.
- In no-improvement rounds, algorithm now attempts deterministic perturb-restart probes around current price, reruns Stage-II selection, refines price, and accepts if epsilon improves.
- This is a focused local escape mechanism (stronger than estimator-only change).

### C. One-to-one run package (single config/seed/domain/users)

Added `scripts/run_stage1_convergence_alignment_package.py`:
- Runs all variants on the **same sampled user instance**.
- Uses a **single shared true-SE oracle** for all variant distances.
- Exports per-variant trajectories and one summary table only (no cross-trial averages).
- Generates one overlay figure with all variant trajectories + true-SE marker.

## 5) Are improvements real?

- **Metric correctness/traceability improvement: YES (clear).**
  - Distances are now strictly aligned to one shared oracle per run.
  - NaN distance progression is blocked by hard checks.
- **Optimization improvement: PARTIAL.**
  - Guided variant now runs more steps in this instance, indicating stronger search behavior.
  - However, guided and neighborhood-only still end at very similar dist-to-SE on this seed (while epsilon differs).
  - This remains evidence of objective mismatch risk between epsilon proxy and price-space SE distance.

## 6) Recommended default + next risks

Recommended default:
- Keep `algorithm_5_stackelberg_guided_search` as default (now with perturb-restart fallback) for Stage-I search robustness.

Remaining risks:
1. **Proxy mismatch risk**: epsilon minimization may not imply improved distance-to-SE.
2. **Seed sensitivity**: single-instance evidence is traceable but not sufficient for global claim.
3. **Oracle cost**: strict SE-distance alignment requires grid-oracle calls; expensive for larger sweeps.

Next step (if publishing convergence claims):
- Run a controlled multi-seed sweep with the same strict alignment protocol and report both epsilon and dist-to-SE distributions (separately, without mixing narratives).
